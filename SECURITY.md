# CryptoKeeper 安全规范

## 加密算法

### 数据加密
- **算法**: AES-256-GCM (Authenticated Encryption)
- **密钥长度**: 256位
- **模式**: Galois/Counter Mode
- **优点**: 提供机密性、完整性认证和防重放攻击

### 密钥派生
- **主密钥派生**: PBKDF2-SHA256 (150,000次迭代)
- **专用密钥派生**: HKDF-SHA256
- **盐值**: 32字节随机生成

## 密钥管理

### 分层密钥体系
```
用户主密码
    ↓
PBKDF2 → 主密钥
    ↓
HKDF → 专用加密密钥
     → 专用认证密钥
     → 密码哈希密钥
```

### 密钥隔离原则
- 不同用途使用独立密钥
- 防止密钥复用导致的安全风险
- 确保前向安全性

## 身份认证

### 主密码验证
- 恒定时间比较 (timingSafeEqual) - 防止时序攻击
- 多因素强度评估:
  - 长度 ≥ 12位
  - 包含大小写字母、数字、特殊符号
  - 非常见弱密码

### 防暴力破解
- 递增锁定策略:
  - 3次失败: 锁定5分钟
  - 4次失败: 锁定10分钟
  - 5次失败: 锁定30分钟

## 内存安全

### 敏感数据保护
- 关键变量内存覆盖清除
- Buffer填充零值
- 垃圾回收提示
- 最小化敏感数据在内存中的驻留时间

## 安全监控

### 实时日志
- 记录所有关键安全事件
- 包括加密、解密、认证尝试等操作
- 支持审计追踪

### 安全统计
- 成功/失败操作计数
- 操作频率分析
- 异常行为检测

## 用户建议

### 创建强密码
✅ 推荐格式: `C0mPlex!Passw0rd@2026` 
❌ 避免格式: `password123`, `123456789`

### 安全实践
- 定期更换主密码
- 启用操作系统级锁屏
- 定期备份重要数据
- 避免在公共计算机上使用
- 使用弱密码模式时请了解风险

## 安全功能

### 弱密码模式
- 用户可选择允许使用弱密码（不推荐）
- 启用时会显示安全警告
- 系统会记录使用弱密码的风险

### 数据统计
- 空间、密码本、密码的数量统计
- 直接从数据库获取准确数据
